Пошаговая инструкция для пользователя
Шаг 1. Подготовка входных данных
Создать или отредактировать файлы types.txt и objects.txt в любом текстовом редакторе (Блокнот, Notepad++ и т.п.) в соответствии с форматами, описанными выше. Файлы должны быть сохранены в кодировке ANSI или UTF-8.
Шаг 2. Размещение файлов
Поместить исполняемый файл reliability.exe и входные файлы (types.txt, objects.txt) в одну директорию. 
Шаг 3. Запуск программы
Дважды щёлкнуть по файлу reliability.exe или запустить его из командной строки.
Шаг 4. Наблюдение за загрузкой данных
Программа выведет на экран сообщения о количестве загруженных типов и объектов:
=== СИСТЕМА МОНИТОРИНГА НАДЁЖНОСТИ ОБОРУДОВАНИЯ ===

Загружено типов: 15
Загружено объектов: 91
Если в процессе загрузки были обнаружены ошибки в формате строк, программа выведет соответствующие предупреждения:
ОШИБКА: objects.txt строка 5: ID типа не существует, пропущено
ОШИБКА: objects.txt строка 12: Неверная дата окончания, пропущено
Шаг 5. Ввод текущей даты
Программа запросит ввод текущей даты:
Введите текущую дату (ДД.ММ.ГГГГ):
Пользователь должен ввести дату в формате ДД.ММ.ГГГГ, например:
10.12.2024
При вводе даты в неверном формате программа сообщит об ошибке и повторит запрос:
Неверный формат даты. Используйте ДД.ММ.ГГГГ
Введите текущую дату (ДД.ММ.ГГГГ):
 
Шаг 6. Ожидание завершения расчётов
После ввода даты программа выполняет все необходимые расчёты. В зависимости от количества объектов (до 1000) это занимает не более 5 секунд. Программа не требует дополнительного взаимодействия с пользователем.
Шаг 7. Получение результата
При успешном завершении расчётов программа выведет сообщение:
Отчёт сохранён в report.txt

Готово.
В той же директории, где находится исполняемый файл, будет создан (или перезаписан) файл report.txt.
Шаг 8. Просмотр отчёта
Открыть файл report.txt в любом текстовом редакторе (Блокнот, Word, Notepad++). Рекомендуется использовать редактор с поддержкой кодировки UTF-8 для корректного отображения русских символов.
Шаг 9. Анализ результатов
На основе данных отчёта пользователь может:
•	определить типы оборудования с превышением допустимого процента брака;
•	оценить прогнозируемое число отказов в ближайшие 3, 6 и 12 месяцев;
•	скорректировать планово-предупредительные ремонты на основе рекомендуемого МРП;
•	выявить оборудование с наименьшей надёжностью для приоритетной замены.
