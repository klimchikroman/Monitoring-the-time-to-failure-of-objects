Формат файла типов оборудования (types.txt)
<id> <name> <daily_hours> <design_lifetime> <design_defect_rate>
<daily_hours> - Сколько часов в день работают объекты данного типа
<design_lifetime> - Расчётный срок службы, нужен для сравнения с данными, полученными статистикой
<design_defect_rate> - Расчётный процент брака (Отказов раньше одного расчётного срока), нужен для сравнения с данными, полученными статистикой

Формат файла объектов оборудования (objects.txt)
<id> <name> <type_id> <status> <start_date> <end_date>
<type_id> - ID Типа, которому принадлежит объект
<status> - Состояние 
      -1 - не эксплуатируется
       0 - эксплуатируется
       1 - снят без отказов
       2 - отказал
<start_date> - дата начала, нужна при статусах 0,1,2
<end_date> - дата прекращения, нужна для статусов 1,2
